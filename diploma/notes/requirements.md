# Реализованные функции

| Функция | Статус | Где реализовано | Комментарий |
|---|---|---|---|
| Запуск через Docker Compose | Реализовано | `docker-compose.yml` | Поднимаются контейнеры приложения и PostgreSQL |
| Сборка контейнера приложения | Реализовано | `Dockerfile` | Multi-stage build: сборка Go-бинарника и запуск в Alpine |
| Подключение к PostgreSQL | Реализовано | `cmd/app/main.go`, `internal/config/config.go` | Используется `DATABASE_DSN` |
| Автоматические миграции | Реализовано | `internal/repository/postgres.go`, `migrations/` | SQL-файлы применяются при старте приложения |
| REST API | Реализовано | `internal/httpapi/server.go` | Импорт, регистрация, выбор, экспорт, health-check |
| Telegram-бот | Реализовано | `internal/bot/bot.go` | Команды студента и администратора |
| Идентификация студента | Частично реализовано | `internal/bot/bot.go`, `internal/repository/postgres.go` | Студент вводит ФИО и группу, запись связывается с Telegram ID |
| Загрузка студентов | Реализовано | `internal/importer/importer.go`, `internal/repository/postgres.go` | CSV/JSON, нормализация ФИО и группы |
| Загрузка дисциплин | Реализовано | `internal/importer/importer.go`, `internal/repository/postgres.go` | CSV/JSON, группировка по коду выбора |
| Просмотр доступных выборов | Реализовано | `internal/bot/bot.go`, `internal/repository/postgres.go` | Выборы фильтруются по группе студента |
| Подача выбора студентом | Реализовано | `internal/bot/bot.go`, `internal/service/service.go`, `internal/repository/postgres.go` | Проверяются дедлайн, группа, варианты и места |
| Факультативы без лимита выбора | Реализовано | `internal/bot/bot.go`, `internal/repository/postgres.go` | `elective` выбираются множественно с подтверждением |
| Однократный обязательный выбор | Реализовано | `internal/repository/postgres.go` | Повторный student-submit для `required_choice` блокируется |
| Модули мобильности | Реализовано | `internal/repository/postgres.go`, `internal/bot/bot.go` | Множественный выбор, проверка суммы з.е. |
| Автоматическое распределение | Реализовано | `internal/repository/postgres.go` | Для `required_choice`, студенты без выбора назначаются на варианты с меньшей заполненностью |
| Ручное изменение выбора администратором | Реализовано | `internal/bot/bot.go`, `internal/service/service.go` | Команда `/set_choice` |
| Экспорт CSV | Реализовано | `internal/service/service.go`, `internal/httpapi/server.go`, `internal/bot/bot.go` | Результаты, студенты, зарегистрированные студенты |
| Экспорт JSON | Реализовано | `internal/service/service.go`, `internal/httpapi/server.go`, `internal/bot/bot.go` | Результаты, студенты, зарегистрированные студенты |
| Экспорт Excel | Не реализовано | Не обнаружено | Можно указать как перспективу развития |
| Генерация Word/DOCX | Реализовано | `internal/docx/docx.go` | Формируется заявление `.docx` |
| Генерация PDF | Не реализовано | Не обнаружено | В `reports/` есть PDF-примеры, но генерации в коде нет |
| Управление администраторами | Реализовано | `internal/bot/bot.go`, `internal/repository/postgres.go` | `/admins`, `/add_admin`, `/remove_admin` |
| Автоматические тесты | Частично реализовано | `internal/importer/importer_test.go`, `internal/repository/rules_test.go` | Проверяются импорт, нормализация группы и правила валидации |

## Функциональные требования по факту реализации

1. Система должна запускаться как серверное приложение Go.
2. Система должна хранить данные в PostgreSQL.
3. Система должна выполнять миграции БД при старте.
4. Система должна импортировать студентов из CSV/JSON.
5. Система должна импортировать выборы и дисциплины из CSV/JSON.
6. Система должна предоставлять Telegram-интерфейс для студентов.
7. Система должна предоставлять Telegram-команды администратора.
8. Система должна предоставлять REST API для проверки и интеграций.
9. Система должна фильтровать доступные выборы по группе студента.
10. Система должна учитывать дедлайн выбора для студенческих заявок.
11. Система должна учитывать лимит мест в варианте дисциплины.
12. Система должна поддерживать типы выбора `required_choice`, `elective`, `mobility`.
13. Система должна экспортировать данные в CSV/JSON.
14. Система должна формировать заявление студента в DOCX.

## Нефункциональные требования по факту реализации

1. Приложение должно запускаться в контейнере Docker.
2. PostgreSQL должен запускаться отдельным контейнером.
3. Конфигурация должна задаваться через переменные окружения.
4. Данные PostgreSQL должны сохраняться в Docker volume.
5. Сервер должен предоставлять endpoint `/health`.
6. Приложение должно корректно завершать HTTP-сервер при сигнале остановки.

## Входные данные

- CSV/JSON со студентами: ФИО, направление, группа.
- CSV/JSON с выборами: код выбора, название, тип, ООП, руководитель, группы, дедлайн, минимум, максимум, дисциплина, места, кредиты, семестр, преподаватель, ссылка.
- Команды Telegram-бота.
- JSON-запросы REST API.

## Выходные данные

- CSV/JSON с результатами распределения.
- CSV/JSON со студентами.
- CSV/JSON с зарегистрированными студентами и выборами.
- DOCX-заявление студента.
- Сообщения Telegram-бота.

## Ограничения

- Excel не реализован.
- PDF не генерируется приложением.
- Автоматическое распределение реализовано только для `required_choice`.
- Приоритеты студента в отдельной структуре не хранятся.
