# Технический анализ проекта

## Назначение

Проект представляет собой клиент-серверное приложение на Go для автоматизации выбора и распределения студентов по дисциплинам по выбору. Основной интерфейс пользователя реализован через Telegram-бота, дополнительный технический интерфейс — через REST API.

## Точка входа

Точка входа находится в `cmd/app/main.go`. При старте выполняются загрузка конфигурации, подключение к PostgreSQL, запуск миграций, инициализация сервиса, запуск HTTP-сервера и, при наличии токена, запуск Telegram-бота.

## Docker и запуск

`docker-compose.yml` содержит два сервиса: `postgres` и `app`. PostgreSQL использует volume `postgres_data`. Приложение собирается из `Dockerfile`, где используется multi-stage build: сначала сборка Go-бинарника, затем запуск в Alpine.

## Конфигурация

Конфигурация загружается из переменных окружения в `internal/config/config.go`. Поддерживаются `HTTP_ADDR`, `DATABASE_DSN`, `TELEGRAM_BOT_TOKEN`, `ADMIN_TELEGRAM_IDS`, `APP_TIMEZONE`, `SEED_ON_START`, `SEED_STUDENTS_FILE`, `SEED_CHOICES_FILE`.

## PostgreSQL и миграции

Работа с PostgreSQL реализована в `internal/repository/postgres.go`. Миграции находятся в `migrations/` и применяются функцией `RunMigrations`. Схема включает студентов, выборы, группы выбора, варианты дисциплин, записи студентов, администраторов и журнал миграций.

## Telegram-бот

Telegram-бот реализован в `internal/bot/bot.go`. Студент может зарегистрироваться, посмотреть доступные выборы, выбрать дисциплины, посмотреть записи и получить заявление. Администратор может импортировать файлы, запускать автоназначение, экспортировать данные, изменять выбор студента и управлять администраторами.

## REST API

REST API реализован в `internal/httpapi/server.go`. Доступны endpoints для health-check, импорта, регистрации, получения выборов и вариантов, подачи выбора, автоназначения, экспорта результатов и генерации DOCX.

## Импорт данных

Импорт CSV/JSON реализован в `internal/importer/importer.go`. Поддерживаются студенты и выборы. Для студентов обязательны ФИО, направление и группа. Для выборов используются код, название, тип, группы, дедлайн, параметры выбора и данные дисциплин.

## Экспорт данных

Экспорт CSV/JSON реализован в `internal/service/service.go`. Формируются результаты распределения, список студентов и список зарегистрированных студентов с выборами. Генерация DOCX реализована в `internal/docx/docx.go`. Excel и PDF не обнаружены.

## Алгоритм распределения

Алгоритм `AutoAssignRequired` реализован в `internal/repository/postgres.go`. Он применяется только к `required_choice`, находит студентов доступных групп без записи по выбранному коду, сортирует варианты по текущей заполненности и назначает студента на первый вариант со свободными местами. Алгоритм не использует приоритеты студента и не заявлен как оптимальный.

## Тесты

Автоматические тесты обнаружены в `internal/importer/importer_test.go` и `internal/repository/rules_test.go`. Покрыты импорт JSON, обработка некорректного формата группы, нормализация группы и правила валидации выбора. Интеграционные тесты Telegram/REST/PostgreSQL не обнаружены.

## Не обнаружено

- Веб-интерфейс.
- Excel-импорт/экспорт.
- PDF-генерация.
- Хранение приоритетов студента.
- Swagger/OpenAPI.
- Интеграционные тесты полного сценария.
