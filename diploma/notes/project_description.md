# Описание проекта

## Тема ВКР

Разработка клиент-серверного приложения для автоматизации распределения студентов по дисциплинам по выбору с использованием языка Go.

## Назначение

Система предназначена для автоматизации процесса выбора студентами дисциплин и последующего распределения студентов по выбранным дисциплинам с учетом ограничений учебного процесса: принадлежности студента к группе, сроков выбора, лимитов мест, типа выбора и количества зачетных единиц для модулей мобильности.

## Пользователи

1. Студент — взаимодействует с системой через Telegram-бота, проходит идентификацию по ФИО и группе, просматривает доступные выборы, выбирает дисциплины и получает заявление в формате DOCX.
2. Администратор — загружает данные студентов и дисциплин, управляет выбором студентов, запускает автоматическое распределение, выгружает результаты и управляет списком администраторов.

## Основной сценарий

1. Администратор загружает список студентов.
2. Администратор загружает список дисциплин и вариантов выбора.
3. Студент открывает Telegram-бота.
4. Студент регистрируется по ФИО и коду группы.
5. Система связывает запись студента с Telegram ID.
6. Студент просматривает доступные для своей группы выборы.
7. Студент выбирает дисциплины.
8. Система проверяет дедлайн, доступность выбора, принадлежность вариантов к выбору и лимиты мест.
9. После дедлайна администратор может запустить автоматическое распределение студентов без выбора для обязательных дисциплин.
10. Администратор выгружает результаты в CSV/JSON.
11. Студент может получить заявление в формате DOCX.

## Что реализовано

- Запуск приложения и PostgreSQL через Docker Compose.
- REST API для импорта, просмотра данных, регистрации, подачи выбора, автоназначения и экспорта.
- Telegram-бот для студентов и администраторов.
- Импорт студентов из CSV/JSON.
- Импорт выборов и дисциплин из CSV/JSON.
- Типы выборов `required_choice`, `elective`, `mobility`.
- Множественный выбор факультативов без ограничений по `min_selected` и `max_selected`.
- Выбор обязательной дисциплины студентом только один раз.
- Автоматическое распределение студентов без выбора для `required_choice`.
- Экспорт результатов, студентов и зарегистрированных студентов в CSV/JSON.
- Генерация заявления DOCX.
- Миграции PostgreSQL при старте приложения.
- Хранение администраторов в PostgreSQL.

## Что не реализовано

- Веб-интерфейс студента или администратора.
- Авторизация по логину и паролю.
- Импорт или экспорт Excel.
- Генерация PDF из приложения.
- Хранение ранжированных приоритетов студента.
- Оптимизационный алгоритм с доказательством оптимальности.
- Swagger/OpenAPI-спецификация.

## Ограничения текущей версии

- Основной пользовательский интерфейс реализован через Telegram-бота.
- Автоматическое распределение применяется только к выбору типа `required_choice`.
- Для модулей мобильности проверяется сумма зачетных единиц, но не реализованы приоритеты.
- Интеграционные тесты с реальной PostgreSQL и Telegram Bot API не обнаружены.
- Экспорт Excel рассматривается как возможное направление развития, но в коде не реализован.
