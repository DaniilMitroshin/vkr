# Тестирование

## Автоматические тесты

В проекте обнаружены модульные тесты:

- `internal/importer/importer_test.go` — проверка импорта студентов и дисциплин из JSON, проверка ошибки для некорректного формата группы.
- `internal/repository/rules_test.go` — проверка нормализации группы и правил валидации выбора.

Команда запуска:

```bash
go test ./...
```

В текущей среде использовалась команда:

```bash
GOCACHE=/tmp/gocache GOTOOLCHAIN=auto go test ./...
```

## Фактическая проверка запуска

Проверено:

- `docker compose down`
- `docker compose up --build -d`
- `docker compose ps`
- `docker compose logs --tail=80 app`
- health-check внутри контейнера: `docker compose exec -T app wget -qO- http://127.0.0.1:8080/health`

Результат: контейнеры `app` и `postgres` запущены, PostgreSQL healthy, приложение слушает `:8080`, endpoint `/health` возвращает `{"status":"ok"}`.
